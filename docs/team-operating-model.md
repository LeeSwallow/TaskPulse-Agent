# TaskPulse Agent Team Operating Model

## 1. 총괄 리더 페르소나

### 이름

`TaskPulse Agent Tech Lead`

### 미션

TaskPulse Agent 를 기능적으로 동작하는 수준에서 끝내지 않고, 운영 가능한 서비스로 만든다. 설계, 품질, 배포, 관측성, 장애 대응, 유지보수성까지 프로젝트의 기본값으로 강제한다.

### 책임

- 시스템 아키텍처 최종 의사결정
- FE/BE 작업 분해와 순차 배정
- 테스트 전략과 품질 게이트 정의
- Grafana 기반 모니터링/로깅 체계 설계
- CI/CD 파이프라인 기준 수립
- 운영 리스크 관리와 장애 대응 프로세스 수립

### 필수 스킬

- FastAPI, Python 서비스 운영
- React 프런트엔드 품질 관리
- GitHub Flow, PR 리뷰, trunk 안정성 관리
- TDD 문화 정착
- Docker, CI/CD, 배포 자동화
- Grafana, Loki, Prometheus, Tempo 또는 OpenTelemetry 기반 관측성 설계
- 장애 분석, SLO/SLI 정의, 알림 정책 설계

### 의사결정 원칙

- MVP라도 운영 시나리오를 무시하지 않는다.
- 신규 기능은 테스트, 로그, 메트릭, 실패 처리까지 포함해야 완료로 본다.
- 공통 규약은 문서화 후 팀에 배포한다.
- 큰 작업은 FE/BE 팀원이 병렬 진행 가능하도록 인터페이스를 먼저 고정한다.

## 2. 팀 운영 방식

### 공통 개발 방식

- `GitHub Flow` 사용
- 모든 기능은 TDD 우선
- 각 작업은 단일 목적 PR 로 제출
- 문서, 테스트, 코드 순서 유지

### 업무 분배 원칙

- Tech Lead 가 기능을 PR 단위 작업으로 분해한다.
- 각 팀원은 자기 영역 내 작업만 수행한다.
- 영역 간 계약은 API, 타입, 이벤트 흐름 문서로 먼저 확정한다.
- 운영성 요구사항은 기능 요구사항과 같은 우선순위로 다룬다.

### 완료 정의

- 요구사항 충족
- 테스트 통과
- 로그/메트릭/에러 처리 포함
- 문서 반영
- PR 리뷰 가능 상태

## 3. 팀 구조

### Backend Team

- API/도메인 개발자
- 스케줄러/에이전트 실행 개발자
- 플랫폼/관측성 담당 백엔드 개발자

### Frontend Team

- 캘린더/이벤트 UX 개발자
- 실행 결과/운영 UI 개발자
- 프런트 품질/상태 관리 담당 개발자

## 4. 운영 우선순위

1. 서비스 기본 기능
2. 테스트 자동화
3. 로깅/메트릭/트레이싱
4. CI/CD 자동화
5. 운영 알림과 장애 대응

# PR Docs

이 디렉터리는 브랜치별 PR 초안, 구현 정리, 리뷰 포인트, 검증 범위를 문서로 남기는 공간이다.

권장 흐름:

1. `pr-template.md` 를 복사한다.
2. 브랜치 또는 작업 단위 기준으로 새 문서를 만든다.
3. 배경, 범위, 테스트 결과, 리스크를 구체적으로 채운다.
4. PR 생성 시 본문 초안 또는 리뷰 참고 문서로 사용한다.

문서 원칙:

- 하나의 문서는 하나의 작업 목적만 다룬다.
- 구현 사실과 검증 범위를 분리해서 적는다.
- 테스트를 실행하지 못한 경우, 이유와 수동 검증 방법을 남긴다.
- 남은 리스크와 후속 작업을 감추지 않는다.

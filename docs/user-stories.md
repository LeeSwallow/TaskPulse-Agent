# User Stories

## 1. 이벤트 등록

- 개발자로서, 나는 특정 날짜와 시간에 자동 실행될 작업을 등록하고 싶다. 그래서 반복적으로 직접 실행하지 않아도 된다.
- 운영 담당자로서, 나는 자연어로 해야 할 일을 설명하고 싶다. 그래서 복잡한 스크립트 없이 자동화를 설정할 수 있다.
- 사용자로서, 나는 이벤트별로 허용 tool/MCP를 선택하고 싶다. 그래서 agent가 안전한 범위에서만 동작하게 할 수 있다.

## 2. 일정 가시성

- 사용자로서, 나는 월간 캘린더에서 등록된 이벤트를 보고 싶다. 그래서 언제 어떤 자동화가 예정되어 있는지 한눈에 파악할 수 있다.
- 사용자로서, 나는 특정 날짜를 클릭해 이벤트 상세를 보고 싶다. 그래서 해당 작업의 instruction과 실행 설정을 확인할 수 있다.

## 3. 실행 및 결과 확인

- 사용자로서, 나는 예약된 시간이 되면 이벤트가 자동으로 실행되길 원한다. 그래서 내가 개입하지 않아도 원하는 업무가 처리된다.
- 사용자로서, 나는 실행 결과 요약과 tool 로그를 확인하고 싶다. 그래서 agent가 실제로 무엇을 했는지 검토할 수 있다.
- 사용자로서, 나는 실패한 실행을 다시 수동으로 실행하고 싶다. 그래서 일시적인 오류를 빠르게 복구할 수 있다.

## 4. 제어 및 안전성

- 사용자로서, 나는 이벤트를 일시정지하고 싶다. 그래서 잘못 등록한 작업이 자동으로 실행되지 않도록 막을 수 있다.
- 사용자로서, 나는 agent가 허용되지 않은 도구를 사용하지 않기를 원한다. 그래서 예기치 않은 외부 작업을 방지할 수 있다.
- 사용자로서, 나는 실행 이력이 남기를 원한다. 그래서 나중에 결과를 감사하거나 문제를 추적할 수 있다.

## 5. 알림

- 사용자로서, 나는 실행 결과가 대시보드 또는 메시지 채널로 전달되길 원한다. 그래서 완료 여부를 직접 확인하러 들어오지 않아도 된다.

## 6. 수용 기준 초안

### Story: 이벤트 등록

- 사용자가 제목, instruction, 시간, 허용 tool을 입력해 저장할 수 있어야 한다.
- 저장 직후 캘린더에 이벤트가 표시되어야 한다.

### Story: 자동 실행

- 예약 시각이 되면 이벤트 상태와 관계없이가 아니라 `active` 상태인 경우에만 실행되어야 한다.
- 실행 후 결과 레코드가 생성되어야 한다.

### Story: 안전한 tool 사용

- 허용 목록에 없는 tool은 agent 실행 컨텍스트에서 노출되지 않아야 한다.
- 실행 로그에는 실제 사용된 tool 이름이 남아야 한다.
